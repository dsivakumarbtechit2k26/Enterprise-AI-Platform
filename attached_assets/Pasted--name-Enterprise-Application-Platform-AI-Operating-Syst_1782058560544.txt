## Multi-Tenant SaaS Platform
### Database Architecture (Central Platform + Tenant Databases)
- **Central Platform Database**: Stores shared data that is common to all tenants: tenant records, subscription plans, billing info, user identities (global logins), roles, permissions, audit logs, notifications, platform configurations, and MCP tool registrations. This database is the single source of truth for subscription billing, authentication, and cross-tenant administration, making installation and maintenance straightforward.
- **Tenant Databases**: Each tenant has its own isolated database (or PostgreSQL schema) that holds all tenant-specific business data: ERP modules (inventory, orders, HR, etc.), dynamic modules, workflows, AI embeddings, and file references. Isolation guarantees data security, performance, and independent backup/restore capabilities. Lower-tier plans may optionally use a shared database with row-level security, but dedicated databases are the default for production.
- **Tenant Resolution**: A middleware identifies the tenant via subdomain or custom domain, then dynamically sets the database connection using Laravel's `config('database.connections.tenant')` pattern or the `stancl/tenancy` package. The connection is resolved once per request and cached for subsequent queries.
- **Migration Strategy**: Central migrations live in the default `database/migrations` and run once during platform installation. Tenant migrations are stored in a dedicated directory (`database/migrations/tenant`) and are applied automatically to each tenant’s database upon creation or upgrade (using `tenancy:migrate` or a custom queue job).
- **Subscription Billing Integration**: The central platform manages plans, coupons, invoices, and payment methods via Laravel Cashier (Stripe/Paddle). Feature gating is enforced by global middleware that checks the tenant’s plan features. Quotas (users, storage, API calls) are tracked centrally; optional usage counters can be synced to tenant databases for faster local checks.
- **Installation & Maintenance Benefits**:
  - A single central dashboard to manage all tenants, plans, and platform settings.
  - Subscription logic and payment webhooks are handled once, not duplicated per tenant.
  - Tenant databases can be hosted on separate servers or clusters for horizontal scaling.
  - New tenant provisioning is fully automated: create tenant record → create empty database → run migrations → seed default data → activate subscription.

### Dynamic CMS Page Builder

## System Operations & Automation
### Database Backup
- Automated backups for Central Platform Database and all tenant databases.
- Scheduled backups using Laravel Task Scheduler (daily/weekly snapshots).
- Backup storage: local disk, cloud (AWS S3, DigitalOcean Spaces, etc.), encrypted and compressed.
- Backup monitoring and failure alerts via email/Slack.
- Point-in-time recovery support using binary logs (MySQL) or WAL (PostgreSQL).
- Retention policies (e.g., keep last 7 daily, 4 weekly, 12 monthly).
- One-click restoration and automated testing of backups.

### Cron Job Automation Flow
- Centralized task scheduling via Laravel Scheduler (single cron entry).
- Queue workers managed by Supervisor, monitored via Horizon.
- Automated tenant provisioning: create database, run migrations, seed, activate.
- Subscription lifecycle management: trial expiration notices, billing retries, grace period handling.
- Periodic cleanup: expired tokens, old audit logs, temp files, soft-deleted records.
- AI/ML pipelines: document ingestion, embedding generation, model retraining scheduled jobs.
- Report generation and distribution (email scheduled reports).
- Workflow triggers and escalations (checking SLA, overdue tasks).
- Health checks and uptime monitoring with automated alerts.
- Backup validation: restore test backups periodically to verify integrity.
